# Migration Plan: Add Tavily as parallel search engine in MultiEngineSearch

**Unit ID:** multi-engine-search-tavily
**Search Provider:** tavily
**Migration Strategy:** additive

## Objective

Integrate Tavily as an additional engine alongside Whoogle, DuckDuckGo, and Bing in the MultiEngineSearch aggregator. Tavily results are fetched when TAVILY_API_KEY is set and merged into the existing deduplication/ranking pipeline.

## Implementation Steps

1. Add tavily-python to requirements.txt and pyproject.toml optional deps
2. In MultiEngineSearch.__init__, read TAVILY_API_KEY from env
3. Implement _search_tavily() method using tavily-python client, normalizing results to the existing {title, url, snippet, source, score} shape
4. In MultiEngineSearch.search(), call _search_tavily() when key is present, append results and record 'tavily' in engines_used
5. In execute_query (query.py), detect TAVILY_API_KEY and offer Tavily as an alternative to the Whoogle endpoint for the --pro path

## Files to Modify

### `src/agent_search/core/multi_search.py`
- Add _search_tavily() method and wire it into the search() orchestration loop alongside existing engines. Gate on TAVILY_API_KEY env var, consistent with existing BING_SEARCH_API_KEY pattern.

### `src/agent_search/cli/commands/query.py`
- Add optional --provider flag (or detect TAVILY_API_KEY) to route single-engine queries through Tavily instead of the default Whoogle endpoint.

### `requirements.txt`
- Add tavily-python package dependency.

### `pyproject.toml`
- Add tavily-python to optional-dependencies or core dependencies list.

## Dependency Changes

- Add tavily-python>=0.3.0 to requirements.txt
- Add tavily-python>=0.3.0 to pyproject.toml optional search extras

## Environment Variable Changes

- Add TAVILY_API_KEY — Tavily API key; when set, enables Tavily engine in multi-search and pro query mode

## Notes

Existing BING_SEARCH_API_KEY, AGENT_SEARCH_ENDPOINT, and all Whoogle/DuckDuckGo/Bing code paths are preserved unchanged. Tavily is strictly additive and only activated when its env var is present.
